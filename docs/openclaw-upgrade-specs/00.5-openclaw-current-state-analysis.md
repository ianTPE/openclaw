# OpenClaw Current State Analysis

## 目的

本文件記錄這套升級 spec 所假設的 OpenClaw 當前能力與可驗證 extension points，作為 fork 與實作的起點。

---

## 參考連結說明

以下各小節引用的 URL 為撰寫時的參考路徑，實際 URL 可能隨 OpenClaw upstream 文件調整而變動。建議在 fork 前以 upstream repo 實際內容為準。

---

## 目前可確認的公開能力

### 1. 現有 memory 預設仍是 Markdown-first

目前官方文件明確描述：

- memory source of truth 是 plain Markdown
- 預設 memory search 針對 `MEMORY.md` 與 `memory/**/*.md`
- `memory_search` 搜尋的是 Markdown chunks

參考：

- https://docs.openclaw.ai/concepts/memory
- https://docs.openclaw.ai/reference/memory-config

### 2. Memory plugin slot 已存在

目前官方文件明確描述：

- active memory plugin 由 `plugins.slots.memory` 決定
- 預設是 `memory-core`
- `memory_search` / `memory_get` 由 active memory plugin 提供

參考：

- https://docs.openclaw.ai/concepts/memory
- https://docs.openclaw.ai/cli/memory
- https://docs.openclaw.ai/plugins

### 3. Context engine slot 已存在

目前官方文件明確描述：

- `plugins.slots.contextEngine` 是 exclusive slot
- plugin 可用 `api.registerContextEngine()` 註冊 context engine
- context engine 參與 ingest / assemble / compact lifecycle

參考：

- https://docs.openclaw.ai/concepts/context-engine
- https://docs.openclaw.ai/plugins
- https://docs.openclaw.ai/plugins/architecture

## 對本 spec 的含義

### 已驗證成立的假設

以下假設可以視為已被官方文件支持：

1. `CtxFST` 可以合理地朝 memory plugin 路線發展
2. 若需要 state-aware context assembly，也可以合理地朝 context engine 路線發展
3. OpenClaw 現有預設 memory 仍偏 chunk-based Markdown retrieval

### 尚未完全驗證的部分

以下事項仍需直接看 upstream repo 實作確認：

1. memory plugin 的實際 SDK contract 細節
2. memory plugin 能否單獨覆蓋完整 indexing lifecycle
3. runtime writeback / session state 更新是否已有可重用 hook
4. memory plugin 與 context engine 之間的資料交換慣例

---

## 實作前建議

在正式 fork 前，建議先做一輪 repo-level code reading，至少確認：

1. 現有 `memory-core` 的模組邊界
2. 現有 retrieval orchestration 實作位置
3. context engine 與 memory plugin 的交會點
4. 現有 SQLite schema 與 migration 組織方式

---

## 結論

這份 spec 對 OpenClaw 的假設不是純猜測。

至少在 plugin slots 這件事上，官方文件已經清楚支援：

- `memory` slot
- `contextEngine` slot

因此：

- 短期先走 `fork + additive integration`
- 中長期再收斂到 `memory plugin + context engine/hook`

是一條合理且有官方 extension model 支撐的路線。
