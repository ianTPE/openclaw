# Phase 6 Validation Smoke Tests

Quick-reference command list for validating Runtime State against the acceptance
criteria in `27-phase-6-validation-checklist.md`.

Two validation surfaces are available: the `openclaw memory state` CLI and
focused unit tests. Use both.

---

## CLI smoke test (end-to-end state flow)

Requires a workspace with at least `retrieval-test.ctxfst.md` indexed.

```bash
# 1. Show empty session state (no state yet)
openclaw memory state show --session test-session

# 2. Precheck before preconditions are met → exits 1, prints missing
openclaw memory state precheck --session test-session --entity entity:analyze-resume

# 3. Simulate upload: manually write state:resume-uploaded
openclaw memory state apply-success --session test-session --entity state:resume-uploaded

# 4. Precheck again → should pass now
openclaw memory state precheck --session test-session --entity entity:analyze-resume

# 5. Simulate success: write postconditions for analyze-resume
openclaw memory state apply-success --session test-session --entity entity:analyze-resume

# 6. Show final state: active_states should include postconditions
openclaw memory state show --session test-session

# 7. JSON output for inspection
openclaw memory state show --session test-session --json
```

## Minimal 5-command smoke check

The fastest way to validate Phase 6 end-to-end:

```bash
# 1. Empty session → Missing Preconditions appears
pnpm test -- extensions/memory-core/src/memory/retrieval/session-context.test.ts \
  -t "empty session"

# 2. Preconditions satisfied → Missing Preconditions disappears
pnpm test -- extensions/memory-core/src/memory/retrieval/session-context.test.ts \
  -t "preconditions satisfied"

# 3. Completed entity excluded from next actions
pnpm test -- extensions/memory-core/src/memory/retrieval/session-context.test.ts \
  -t "exclude the completed"

# 4. Success writeback adds state:resume-parsed
pnpm test -- extensions/memory-core/src/memory/runtime/world-state.test.ts \
  -t "adds state:resume-parsed"

# 5. Failure writeback → blocked_by set, no postconditions
pnpm test -- extensions/memory-core/src/memory/runtime/world-state.test.ts \
  -t "blocked_by"
```

All 5 must pass. If any fail, check the corresponding checklist section (A-G)
in `27-phase-6-validation-checklist.md`.
