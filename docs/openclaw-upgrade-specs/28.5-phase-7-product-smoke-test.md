# Phase 7 Product Smoke Test

Quick manual smoke test for validating that the planner behaves correctly in a
real session flow, beyond unit/integration coverage.

This document complements:

- `28-phase-7-validation-checklist.md`
- `27.5-phase-6-smoke-tests.md`

---

## Goal

Validate the product-level claim of Phase 7:

> The same user query should produce different next-step recommendations when
> session world state changes.

This is the practical difference between:

- static graph query: "what is related?"
- state-aware planner: "what should I do next now?"

---

## What To Prove

All of the following should be true:

1. Same query, different session state, different answer
2. Completed steps are not repeatedly recommended
3. Recommended next action respects preconditions
4. Planner output is explainable in terms of current state

---

## Preconditions

Before running this smoke test:

1. Phase 1-7 automated validation is already green
2. `retrieval-test.ctxfst.md` is indexed in workspace memory
3. `openclaw memory state` CLI is available
4. Planner CLI or equivalent preview flow is available in the current build

If planner output is not yet exposed via a stable CLI, use the closest
session-aware planner preview command available in the branch under test.

---

## Canonical Scenario

Use the resume workflow:

- `Upload Resume`
- `Analyze Resume`
- `Generate Report`

Relevant states:

- `state:resume-uploaded`
- `state:resume-parsed`
- `state:analysis-complete`

Use one stable session id throughout:

```bash
SESSION=phase7-smoke
```

---

## Step 1: Empty Session

Start from an empty session:

```bash
openclaw memory state show --session "$SESSION"
```

Ask the planner:

```bash
openclaw memory planner --session "$SESSION" "What should I do next?"
```

Expected behavior:

- Output should not behave as if upload already happened
- Recommendation should be an earliest valid step such as `Upload Resume`
  or clearly indicate that `resume-uploaded` is missing

Failure signs:

- Recommends `Analyze Resume` immediately
- Recommends `Generate Report` immediately
- Gives the same answer you get after later steps are completed

---

## Step 2: Simulate Upload Completed

Write the first success state:

```bash
openclaw memory state apply-success \
  --session "$SESSION" \
  --entity entity:upload-resume
```

Inspect state:

```bash
openclaw memory state show --session "$SESSION"
```

Ask the planner again:

```bash
openclaw memory planner --session "$SESSION" "What should I do next?"
```

Expected behavior:

- Should no longer recommend `Upload Resume`
- Should now prefer `Analyze Resume`
- Explanation should be consistent with `state:resume-uploaded` being active

Failure signs:

- Still recommends `Upload Resume`
- Skips straight to `Generate Report`
- Output is identical to Step 1

---

## Step 3: Simulate Analysis Completed

Write the second success state:

```bash
openclaw memory state apply-success \
  --session "$SESSION" \
  --entity entity:analyze-resume
```

Inspect state:

```bash
openclaw memory state show --session "$SESSION"
```

Ask the planner again:

```bash
openclaw memory planner --session "$SESSION" "What should I do next?"
```

Expected behavior:

- Should no longer recommend `Analyze Resume`
- Should prefer `Generate Report`
- Recommendation should be justified by `state:resume-parsed`

Failure signs:

- Repeats `Analyze Resume`
- Recommends an action whose preconditions are not met
- Output is still identical to Step 2

---

## Step 4: Optional Final Completion

Apply final workflow completion:

```bash
openclaw memory state apply-success \
  --session "$SESSION" \
  --entity entity:generate-report
```

Ask once more:

```bash
openclaw memory planner --session "$SESSION" "What should I do next?"
```

Expected behavior:

- Planner should either report no obvious next action
- Or recommend a later valid step, but not re-suggest already completed steps

---

## Pass Criteria

Phase 7 product smoke test passes if:

1. Step 1, Step 2, and Step 3 produce meaningfully different recommendations
2. Recommendations move forward with the workflow
3. Previously completed actions stop reappearing as the primary next action
4. Output stays aligned with active states and missing preconditions

---

## Minimal Evidence To Capture

Save or paste the output of:

```bash
openclaw memory state show --session "$SESSION"
openclaw memory planner --session "$SESSION" "What should I do next?"
```

for:

- empty session
- after upload success
- after analyze success

That is enough to prove product-level state-sensitive planning behavior.
